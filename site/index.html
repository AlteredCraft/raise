The Principles of RAISE

01

The Human is the Architect

AI is a tool for implementation, not the "contributor".

02

Review is Mandatory

No AI-generated code enters the codebase without human review. "LGTM" is insufficient; "I understand this" is the standard.

03

Own the Code

If you commit AI-generated code, you own it. "The AI wrote it" is never a valid excuse for a bug, security flaw, or logic error.

04

Transparency is Professionalism

We openly disclose which tools were used to generate code. Hiding the use of AI is a breach of trust.

05

Testing is the Truth

AI-generated code requires higher test coverage, not lower. Ideally, tests should be written by the human before generation (TDD).

06

Combat Bloat

We use AI to refactor and simplify, not to add unnecessary verbosity. We respect our teammates' time by aggressively pruning AI output.

07

Security by Design

We actively probe AI suggestions for known vulnerabilities (e.g., injection flaws, hallucinated dependencies) before implementation.

08

Skill Preservation & Growth

We use AI to explain complex concepts, not to bypass learning them. If you cannot explain the code you submitted, you did not write it.

09

Sustainable Pace

We do not use AI to create impossible deadlines. We use efficiency gains to improve code quality and work-life balance.

10

Ethical Sourcing

We prefer AI tools trained on ethically sourced data and respect the licenses of open-source code.

11

Continuous Adaptation

We continuously update our practices as AI models evolve, remaining skeptical of hype and focused on engineering fundamentals.

About RAISE

RAISE is an effort, similar to The Agile Manifesto, to provide a community-derived framework for utilizing AI in the software development process.

This framework addresses the phenomenon known as "The Quality Floor Collapse" and provides a viable, effective middle ground between the extremes of AI rejection and uncritical acceptance.
